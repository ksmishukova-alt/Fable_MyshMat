# Публикация МышМат 2.0 (GitHub + Vercel)

## Шаг 1. GitHub — один раз

1. На github.com создай **пустой** репозиторий (например `MyshMat2`), без README.
2. В папке проекта запусти:

```
setup-git.bat https://github.com/ksmishukova-alt/MyshMat2.git
```

(подставь свой адрес). Скрипт сам починит битый `.git`, закоммитит и запушит.
Дальнейшие обновления — просто `push.bat`.

## Шаг 2. Vercel — один раз

1. vercel.com → **Add New → Project** → Import репозитория `MyshMat2`.
2. **Root Directory: `apps/web`** (кнопка Edit рядом с полем) — обязательно!
3. Environment Variables — добавить (значения из `apps/web/.env.local`):

| Имя | Значение |
|-----|----------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://qptemtnrjtmmwvgtyqhi.supabase.co` |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `sb_publishable_…` (из .env.local) |
| `SUPABASE_SERVICE_ROLE_KEY` | `sb_secret_…` (из .env.local) |
| `AUTH_JWT_SECRET` | из .env.local (или любая длинная случайная строка) |
| `TELEGRAM_BOT_TOKEN` | из .env.local |
| `TELEGRAM_PARENT_CHAT_ID` | `881071570` |
| `TELEGRAM_METHODIST_CHAT_ID` | `881071570` |

(ЮKassa-ключи добавишь, когда будут: `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`.)

4. **Deploy**. Дальше каждый `push.bat` пересобирает сайт автоматически.

## Шаг 3. После первого деплоя

- `https://<домен>/api/health` → должно быть `"db":"supabase"`.
- Войти методистом → «Банк задач» → **«Загрузить недели 1–10 в БД»** (если ещё не нажимала).
- Вебхук ЮKassa (когда появятся ключи): ЛК ЮKassa → HTTP-уведомления →
  `https://<домен>/api/yookassa/webhook`, событие `payment.succeeded`.
