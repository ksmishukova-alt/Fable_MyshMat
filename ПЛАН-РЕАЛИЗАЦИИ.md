# МышМат 2.0 — план реализации (Fable5_Myshmat)

> Согласованные решения: новый проект здесь; объём — все фазы 0–4; Supabase сразу
> (ключи получены, URL уточняется; без БД — авто-fallback на моки); главная — тот же вид,
> чистый код; методист заводит все аккаунты; демо-банк олимпиадных задач генерируется;
> TG-ключи получены, ЮKassa — заглушка до ключей; дуэли — все 3 игры + лидерборд; арт — SVG;
> листочек L3 — методист, остальное авто; деплой GitHub + Vercel.
> **Модель уровней: 3** — L1 тренировка · L2 решение с поддержкой · L3 самостоятельное;
> алгебраический метод — опциональный блок внутри темы (флаг), не уровень.
> **Анимации обязательны**: интерфейс должен ощущаться готовым продуктом.

## 1. Архитектура

```
Fable5_Myshmat/
├── apps/web/                 Next.js 15 (стабильная) + React 19 + TypeScript
│   ├── src/app/              страницы (child / parent / methodist) + API-роуты
│   ├── src/components/       экраны и раннеры
│   ├── src/lib/repo/         репозиторий-слой: интерфейсы + SupabaseRepo + MockRepo
│   ├── src/lib/              auth (JWT httpOnly), сессии, csv, telegram, yookassa
│   ├── src/styles/           дизайн-токены + @layer, без !important
│   └── src/types/            доменные типы (уровни L1–L6, темы, награды, дуэли)
└── packages/db/              SQL-миграции + сиды (children, parents, topics, problems,
                              attempts, rewards, plans, payments, duel_results …)
```

Принципы кода: дизайн-токены из текущей палитры (синий маскот), CSS-слои
(@layer base/components/utilities), container queries по стандарту, никаких `!important`,
тач-цели ≥44px, контраст ≥4.5:1, prefers-reduced-motion (правила ui-ux-pro-max).

**Анимации (обязательно, уровень готового продукта):** появление карточек и экранов
(stagger fade/slide), микровзаимодействия кнопок и статусов, анимированный маскот
(idle-дыхание, реакции на верно/неверно, «ведёт» по шагам), открытие сундука,
получение звёзд/наклеек (полёт к счётчику), прогресс-бары с easing, переходы карты мира,
конфетти при переводе на уровень; всё уважает prefers-reduced-motion.

## 2. Роли и вход
- Ребёнок: логин + PIN (bcrypt), видит только детские экраны.
- Родитель / методист: email + пароль. Самореги нет — методист создаёт всех в кабинете.
- JWT в httpOnly cookie, middleware разделяет зоны /child /parent /methodist.

## 3. Детская зона
- **Главная** — пиксельно близко к текущей (грид logo/profile/hero/daily/side/route/world),
  переписанная чисто.
- **Daily**: 10 существующих раннеров переносятся как есть; контент недель 1–10 → БД;
  расписание по датам от методиста; 3 попытки; submitted → МышРутка → сундук.
- **Олимпиадный мир (ядро)**: карта тем с зависимостями; тема = один узел с глубиной L1–L3.
  Раннеры-под-тему: **L1 тренировка** — пошаговый мастер с подсказками (типовые и нетиповые);
  **L2 с поддержкой** — сжатые шаги: собрать план, выбрать действие, записать выражение,
  найти ошибку; **L3 самостоятельное** — листочек (фото) + авто-проверка ответа.
  Опциональный блок «алгебраический метод» внутри темы (флаг). ~10 задач на уровень,
  4 подряд без ошибок → уровень выше, каскад провала: подсказки → альтернативная задача →
  откат → TG методисту.
- **Демо-банк**: 4 темы («Головы и ноги», чётность, логика, Дирихле) × L1–L3.
- **Награды**: звёзды-валюта, ежедневный сундук (наклейка/звёзды/приз), значки за темы,
  карта мышления, альбом наклеек, лавка, маскот-тамагочи с SVG-нарядами, загадка дня.
- **Дуэли**: устный счёт, закономерности, быстрая логика + таблица лидеров.

## 4. Кабинет методиста
Создание детей/родителей и выдача доступов; сборка плана из шаблона + индивидуальная
корректировка; банк задач (создание/редактирование + CSV-импорт Daily и олимпиады);
проверка ручных заданий (аудио, чтение, фото листочка L5); аналитика: все попытки,
ошибки, тайминги, застревания.

## 5. Кабинет родителя
Сводный отчёт (время, темы, частые ошибки, рекомендации), несколько детей,
расписание Daily, подписка/оплата ЮKassa.

## 6. Интеграции
- **Supabase**: Postgres + Storage (фото, аудио); без ключей — MockRepo.
- **ЮKassa**: подписки родителя (создание платежа, webhook), рубли.
- **Telegram-бот**: методисту — застревания; родителю — мягкий прогресс.
- Всё за env-переменными в `.env.local` (не коммитится).

## 7. Порядок работ
1. Каркас: monorepo, токены/стили, типы, репозиторий-слой, миграции БД.
2. Auth + три зоны + главная (чистый ремейк).
3. Daily из БД + перенос раннеров и контента + МышРутка/сундук.
4. Олимпиадное ядро: уровни, раннеры-под-тему, демо-банк, карта мира.
5. Награды: звёзды, лавка, наклейки, значки, тамагочи, загадка дня.
6. Кабинет методиста (план, банк, CSV, проверка, аналитика).
7. Кабинет родителя + отчёты.
8. Дуэли + лидерборд.
9. ЮKassa + Telegram.
10. Верификация: сборка, типы, сквозной прогон вход→Daily→МышРутка→тема→уровни→награды;
    git-обвязка (README, bat-скрипты), инструкция деплоя Vercel.

## 8. CSV-форматы (предложение)
- Daily: `subject,week,day,runner,title,payload_json,answer_json,hints`
- Олимпиада: `topic,level,order,statement,image,actions_count,answer,steps_json,hints_json,algebra_flag,reward`
